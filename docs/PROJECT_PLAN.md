# MuseLens 实施计划

## 项目问题

解决个人图片数量增多后依赖文件名和文件夹难以查找的问题。系统将文字和图片映射到统一向量空间，提供自然语言搜索、以图搜图、相似图片与重复图片整理。

## 里程碑

1. 工程骨架：CLIP 适配器、向量索引、API 和测试。
2. 持久化：图片元数据、文件哈希和索引恢复。
3. 检索基线：英文文本搜图、以图搜图、余弦相似度。
4. 检索增强：中文查询、多模型对比、重排序与重复检测。
5. 评测体系：Recall@K、MRR、端到端延迟、索引大小和失败案例。
6. 产品化：React 图片瀑布流、后台任务、Docker、CI 和演示视频。

## 完成定义

- 新电脑能按 README 建立图片索引并启动搜索服务。
- 检索实验数字来自可重复运行的脚本，不手工编造。
- API 有输入校验和自动化测试。
- README 展示架构、指标、失败案例、性能和局限性。
- 作者能解释 CLIP、向量归一化、余弦相似度、Recall@K 和索引取舍。

## 2026 项目实践借鉴

- 模型、索引、业务服务和 API 分层。
- 模型延迟加载，避免服务启动时发生隐式下载。
- 将检索质量、延迟与失败案例作为作品主体。
- 通过统一接口替换内存索引、FAISS 或 Qdrant。
