# 开源项目调研与借鉴记录

这份文档记录 MuseLens 参考了什么、为什么参考，以及哪些内容明确不复制。它既是工程决策记录，也用于面试时证明项目不是简单套壳。

## 参考项目

### Immich

- 仓库：<https://github.com/immich-app/immich>
- 重点：本地/自托管图片库、CLIP 智能搜索、缩略图、后台任务、重复检测和大规模虚拟滚动。
- 与 MuseLens 直接相关的问题：智能搜索天然会按相关度返回候选集；维护者也指出，不同查询的有效相似度分布不同，单一固定阈值并不可靠。
- 许可证：AGPL-3.0。

### PhotoPrism

- 仓库：<https://github.com/photoprism/photoprism>
- 重点：语义搜索与日期、颜色、地点、质量等结构化过滤组合；近似重复图片检测；本地优先的相册管理。
- 许可证：AGPL-3.0。

### Ente

- 仓库：<https://github.com/ente-io/ente>
- 重点：隐私优先、后台上传与索引、跨端状态一致性，以及面向真实用户的错误恢复。
- 许可证：AGPL-3.0。

## MuseLens 的工程决策

| 成熟项目中的做法 | MuseLens 的实现策略 | 面试价值 |
| --- | --- | --- |
| 智能搜索返回大量排序结果 | 先实现最低分数与相对分差过滤，再用评测集校准动态拒答 | 能解释固定阈值为何会失败 |
| 原图与缩略图分离 | 生成版本化缩略图，列表只加载缩略图，灯箱再读取原图 | 展示性能与存储设计 |
| 导入和模型推理走后台任务 | 增加任务表、进度接口和可重试状态机 | 展示异步系统与可靠性 |
| 精确哈希去重 | SHA-256 保留，再加入感知哈希识别压缩、缩放后的近似重复 | 展示算法与工程结合 |
| 语义与元数据组合查询 | 增加日期、文件类型和尺寸过滤，再与向量相似度融合 | 展示混合检索能力 |
| 大图片库使用分页/虚拟滚动 | API 使用游标分页，前端使用虚拟列表 | 展示规模化意识 |
| 模型会升级 | 为向量记录模型名、维度和版本，支持重建索引 | 展示数据迁移意识 |

## 搜索质量路线

当前的“绝对分数下限 + 与最高分的相对差”只是 v0，解决了任意输入都显示同一批图片的问题。下一阶段不凭感觉改阈值，而是：

1. 建立中文、英文、无关查询三类测试集。
2. 记录每类查询的正样本、最高分、分数间隙和返回数量。
3. 对比固定阈值、相对分差、Top-K 和基于分布的动态阈值。
4. 用 Recall@K、MRR、无关查询拒答率和查询延迟选择策略。

这会让 MuseLens 与普通“调用 CLIP 然后排序”的课程项目拉开差距。

## 代码与许可证边界

- 可以学习架构、接口思想、公开讨论中的失败案例和产品交互。
- 不复制项目名称、品牌视觉、大段源码或未理解的实现。
- 上述三个参考项目均采用 AGPL-3.0；直接合并其源码可能带来相应的开源义务，因此 MuseLens 采用独立实现。
- 如果未来确实复用第三方代码，必须在提交前确认许可证兼容性，并在 `THIRD_PARTY_NOTICES.md` 中记录来源、版权和修改内容。

## 实施优先级

1. 缩略图缓存与原图分离。
2. 后台索引任务、进度展示和失败重试。
3. 可复现的中文/无关查询评测与动态拒答。
4. 感知哈希近似去重。
5. 元数据过滤、游标分页和虚拟滚动。
