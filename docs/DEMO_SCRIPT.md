# MuseLens 60 秒演示录制脚本

目标：让观看者在一分钟内确认 MuseLens 不是固定关键词页面，而是完整的上传、向量索引和
多模态检索系统。

仓库首页已经提供一个由真实线上 API 结果生成的 15 秒自动播放 GIF。需要更新它时运行：

```bash
python scripts/build_demo_gif.py
```

下面的脚本用于进一步录制带旁白的 60 秒 MP4 版本。

建议录制 1920×1080、30 fps，浏览器缩放 90%～100%，关闭通知并隐藏个人书签。只使用无
隐私内容的测试图片。

| 时间 | 画面操作 | 旁白 / 字幕 |
| ---: | --- | --- |
| 0–5 秒 | 打开 <https://sinbaby-muselens.ms.show>，停留在首页 | “MuseLens 是本地优先的多模态图片检索系统。” |
| 5–13 秒 | 输入 `狗`，展示搜索结果 | “图片不依赖文件名，而是由 SigLIP2 将中文、英文和图片映射到同一语义空间。” |
| 13–20 秒 | 输入 `a person holding a mobile phone` | “它也支持英文长句检索，并明确把分数解释为查询内相对相关性，而不是概率。” |
| 20–29 秒 | 切换到临时图库，选择三张不同内容的测试图上传 | “为了证明后端不是固定图库外壳，访客可以上传自己的图片。” |
| 29–39 秒 | 等待索引完成，使用与文件名无关的中文描述搜索 | “服务端真实完成上传、批量编码、向量索引和检索。” |
| 39–47 秒 | 打开一张结果，再返回并点击“立即清除” | “每位访客的数据按会话隔离，可主动清除，并在 30 分钟后自动过期。” |
| 47–55 秒 | 切换到 GitHub README 的核心结果表 | “完整线上合同包含 84 条双语查询，Hit@5 为 95.24%。” |
| 55–60 秒 | 展示架构图与 GitHub Actions 绿色状态 | “前后端、部署和真实上传链路都由 CI 自动验证。” |

## 录制前检查

- [ ] 线上 `/health` 返回 `model_loaded: true`。
- [ ] 固定图库查询 `狗` 和 `a person holding a mobile phone` 均有结果。
- [ ] 准备三张内容区别明显、文件名不含查询词的测试图片。
- [ ] 临时图库完成索引后再开始正式录制，避免把冷启动等待录进视频。
- [ ] 录制结尾能看到 GitHub CI 绿色状态和 95.24% 指标。
- [ ] 导出后完整观看一次，确认没有个人路径、令牌、通知或敏感图片。

## GitHub 展示建议

导出文件建议命名为 `muselens-demo-60s.mp4`。GitHub README 不适合直接提交大视频文件，
推荐上传到 GitHub Release 或公开视频平台，然后在 README 的界面截图上添加链接。仓库内
只保留本脚本和必要截图，避免增大 Git 历史。
