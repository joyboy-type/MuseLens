# 自动标签评测

## 目标与实现

MuseLens 不额外下载分类模型。图片导入时已经生成 SigLIP2 向量，自动标签只需预先编码一组
受控英文提示词，再计算图文余弦相似度。最终保存稳定 slug、中文显示名称、原始分数和词表
版本，前端负责展示与组合筛选。

词表包含人物、常见动物、交通工具、食物、场景和活动等 40 个标签。每张图片最多保留 3 个，
阈值为最高分相对差 0.04 和绝对下限 0.05。系统允许无标签结果，不把最高候选伪装成必然正确。

## 真实演示集结果

评测使用公开演示图库的 24 张 COCO 2017 图片及其对象类别。场景和风格标签没有对应 COCO
标注，因此精确率与召回率只统计能与 COCO 类别一一对应的受控对象标签。

| 指标 | 结果 |
|---|---:|
| 成功生成标签 | 24 / 24 |
| 至少命中一个已标注对象 | 16 / 24（66.7%） |
| 对象标签精确率 | 84.6% |
| 对象标签召回率 | 38.6% |
| 对象标签 F1 | 53.0% |

低召回主要来自一张图可能包含多个 COCO 对象，而产品最多展示 3 个标签，并且场景/风格标签也
会占用名额。当前选择精确率优先，避免智能相册被大量错误标签污染。标签不是概率，前端不会把
余弦分数显示为“置信度百分比”。

## 可复现性

```bash
python scripts/evaluate_auto_tags.py
```

加入 `--persist` 会把结果写入目标 SQLite；脚本只读取已有向量，不下载图片或复制数据集。
机器可读报告位于 `artifacts/evaluations/auto-tags-demo-v1.json`。

## 后续边界

- 扩展词表前必须重新评测，避免类别之间互相竞争导致阈值漂移。
- 用户手动修正标签、层级相册和更大独立标注集尚未实现。
- 若未来图库规模和标签需求显著增长，再评估专用多标签模型；当前没有数据支持增加第二个常驻模型。
