# CLIP 零样本文本搜图基线

## 实验设置

- 模型：`openai/clip-vit-base-patch32`
- 设备：Apple M4，PyTorch MPS
- 数据：Flickr8k test split 固定抽样100张图片
- 查询：每张图片5条人工描述，共500条英文查询
- 相关项：每条查询对应原始图片
- 相似度：L2归一化后的点积，即余弦相似度

## 结果

| 指标 | 数值 |
|---|---:|
| Recall@1 | 0.852 |
| Recall@5 | 0.976 |
| Recall@10 | 0.998 |
| MRR | 0.908 |
| 平均排名 | 1.356 |
| 中位排名 | 1.0 |

本次运行中，图片编码平均约18.1毫秒/张，文本编码平均约2.82毫秒/查询。该数字受机器状态、批大小和首次运行影响，后续需要多次预热测量并报告均值及波动。

## 局限性

- 候选集只有100张图片，任务明显比真实图片库简单。
- 查询均为英文，尚未评价中文搜索。
- 样本是按文件名固定截取，不一定代表完整数据分布。
- 尚未测量磁盘读取、HTTP请求和持久化索引的端到端延迟。
- 当前结果不能用于声称系统在大型真实图片库上达到同等水平。

## 下一组实验

1. 保存图片向量，避免每次启动重复编码。
2. 记录每条查询的排名，分析 Recall@1 失败案例。
3. 建立中文查询集并比较原始 CLIP 与多语言模型。
4. 将候选集扩展到500或1,000张，观察指标变化。
5. 比较暴力检索与 FAISS/Qdrant 的结果一致性和延迟。
