# 近似重复图片检测评测

## 结论

MuseLens 已实现两层重复检测：导入时用 SHA-256 跳过字节完全相同的文件，导入后用 64 位
感知哈希识别经过缩放、压缩、模糊或曝光调整的近似副本。感知哈希同时使用归一化平均色彩
作为约束，降低结构接近但颜色完全不同图片的误判。

在 500 张 COCO 原图和 2,500 张确定性扰动图上，JPEG 强压缩、模糊、曝光变暗和低分辨率
四类的原图—扰动图直接召回均为 **100%**。全部分组中没有出现跨原图混组，严格分组精确率
为 **100%**。68% 中心裁剪改变了大部分构图，当前规则没有识别，并被保留为明确边界。

## 产品行为

- 旧图库启动后按需补算感知指纹，不需要重新执行模型编码。
- “重复照片”面板按组展示缩略图、分辨率、文件大小和指纹距离。
- 每组默认建议保留像素数更高的版本；像素数相同时优先文件更大的版本。
- 预计可释放空间只计算非推荐保留项的导入副本大小。
- 删除接口只在 `local` 模式开放，只删除 MuseLens 专用图库中的副本和缩略图；源文件路径
  从未写入数据库，因此接口无法删除用户原始照片。
- 公开演示和临时图库可以检查重复组，但不提供逐张删除；临时图库仍通过会话清除或 TTL
  统一销毁。

## 评测协议

数据复用以图搜图鲁棒性实验的确定性 COCO 500 集合：

- 原图：500 张 COCO 2017 validation 图片；
- 每张原图生成 5 个版本：中心裁剪 68%、JPEG quality 24、Gaussian blur 3.2、亮度
  48%、长边降到 96 px 后放大；
- 总候选：3,000 张；
- 默认阈值：感知哈希汉明距离不超过 8，归一化色彩距离不超过 45；
- 正例定义：来自同一原图的任意两个版本；
- 误组定义：一个输出组中出现两个不同 COCO 原图来源。

机器可读结果位于
`artifacts/evaluations/near-duplicate-coco500-v1.json`。

## 结果

| 指标 | 结果 |
| --- | ---: |
| 原图 | 500 |
| 扰动图 | 2,500 |
| 检出的重复组 | 500 |
| 跨来源混组 | **0** |
| 成对精确率 | **100%** |
| 成对召回率 | 66.67% |
| 成对 F1 | 80.00% |
| 最大组大小 | 5 |

成对召回率把同一来源的原图与全部 5 种扰动，以及各扰动之间的所有组合都视为正例。中心
裁剪版本没有进入分组，因此这一严格指标为 66.67%，不能只报告四个容易类别的 100%。

## 性能与实现

3,000 张图片在 M4 上计算指纹约 3.51 秒（约 855 图/秒），已有指纹的分组只需约
0.039 秒。数据库保存指纹，因此用户打开重复检查时不会重新解码整图库。

初始精确两两比较在 5,000 个随机指纹上约需 2.3 秒。最终实现将 64 位哈希拆成
`阈值 + 1` 个块：若两个哈希距离不超过阈值，至少有一个块完全相同。系统仅对共享块的
候选执行精确汉明距离和色彩复核；相同规模实测约 0.066 秒，仍保持精确候选召回。

## 为什么不支持大幅裁剪

感知哈希描述整幅画面的低频结构，大幅裁剪会改变全局构图。盲目提高汉明阈值虽然能召回
更多裁剪图，也会增加不同照片误组。MuseLens 当前定位为高精度清理工具，因此优先避免
误删风险。未来若支持裁剪副本，应增加局部特征匹配或多裁剪指纹，并重新建立带负例的独立
阈值评测，而不是直接放宽现有规则。

## 复现

先按以图搜图评测生成数据，再运行：

```bash
python scripts/evaluate_image_retrieval.py --count 500 --batch-size 16
python scripts/evaluate_duplicates.py
```
