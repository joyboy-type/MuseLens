# 以图搜图鲁棒性评测：COCO 500

## 结论

MuseLens 已在 Apple M4 16 GB 上完成 500 张 COCO 2017 原图、2,500 张扰动查询图的
真实 SigLIP2 编码和检索评测。总体 Recall@1 为 **99.36%**，Recall@5 为 **99.96%**，
MRR 为 **0.9964**。这证明当前以图搜图对常见压缩、亮度、模糊、降采样和中心裁剪具有
很强的鲁棒性。

评测同时支持了一个产品调整：图片搜索使用独立的相对分差 `0.05`。相较原先沿用文本
搜索的 `0.035`，策略后 Recall@5 从 99.80% 提升到 99.96%，平均返回数仅从 1.05 增至
1.11。文本搜索仍保留原阈值，两种模态不再盲目共用参数。

## 数据与协议

- 原始数据：COCO 2017 validation 的 5,000 张本地图片
- 图库：用固定随机种子 `20260718` 选取 500 张
- 查询：每张原图生成 5 个独立版本，共 2,500 张
- 模型：`google/siglip2-base-patch16-224`
- 设备：Apple M4 / MPS
- 相关项定义：扰动查询对应的原始图片
- 候选集：同一批 500 张原图

五种查询扰动：

1. 中心裁剪到原画面的 68%，再恢复到原尺寸；
2. JPEG quality 24 强压缩；
3. Gaussian blur radius 3.2；
4. 亮度降到原图的 48%；
5. 长边降到 96 px 后再放大。

生成数据独立保存在 `data/evaluation/image-retrieval-coco500-v1/`，占用约 114 MB，
不会修改 COCO 原图，也不会提交到 Git。机器可读结果位于
`artifacts/evaluations/image-retrieval-coco500-v1.json`。

| Recall@1 | 99.36% |

## 分扰动结果

| 扰动 | Recall@1 | Recall@5 | MRR |
|---|---:|---:|---:|
| 中心裁剪 | 99.40% | 99.80% | 0.9963 |
| JPEG 强压缩 | 100.00% | 100.00% | 1.0000 |
| 模糊 | 98.60% | 100.00% | 0.9923 |
| 变暗 | 100.00% | 100.00% | 1.0000 |
| 低分辨率 | 98.80% | 100.00% | 0.9932 |

模糊和低分辨率是本协议中最困难的两类，但它们的 Recall@5 仍为 100%。中心裁剪产生了
唯一一个 Top-5 失败样本，该样本最终排第 7。

## 相对分差对比

| 相对分差 | 策略后 Recall@5 | 平均返回数 | P95 返回数 | 空结果率 |
|---:|---:|---:|---:|---:|
| 0.015 | 99.64% | 1.01 | 1 | 0% |
| 0.025 | 99.76% | 1.02 | 1 | 0% |
| 0.035 | 99.80% | 1.05 | 1 | 0% |
| **0.050** | **99.96%** | **1.11** | **2** | **0%** |
| 0.075 | 99.96% | 1.31 | 3 | 0% |

`0.075` 没有带来额外 Recall 收益，只增加了弱结果，因此选择 `0.05`。

## 性能

| 项目 | 结果 |
|---|---:|
| 模型冷加载 | 14.79 s |
| 图库编码 | 14.05 s |
| 单张图库编码 | 28.09 ms |
| 查询编码 | 63.59 s |
| 单张查询编码 | 25.44 ms |
| 2,500 × 500 矩阵检索 | 43.73 ms |
| 单查询纯矩阵检索 | 0.0175 ms |

产品请求还包含图片上传、解码和 HTTP 开销，所以不能把 0.0175 ms 当作端到端延迟；
实际用户体验主要由约 25 ms 的模型编码和请求传输决定。

## 适用边界

这是“同一图片经过常见变化后能否找回原图”的鲁棒性测试，不等同于主观的“不同照片
是否内容相似”。COCO 中可能有多张合理近邻，但本协议只把查询对应原图视为正确。因此，
这组数据适合做版本回归和阈值选择，不能被包装成所有视觉搜索场景的 99.36% 准确率。

## 复现

```bash
python scripts/evaluate_image_retrieval.py --count 500 --batch-size 16
```

再次运行会复用已生成的 2,500 张查询图；使用 `--regenerate` 可重新生成。
