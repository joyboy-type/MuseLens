# MuseLens v0.1.0 面试备忘录

## 30 秒版本

MuseLens 是我独立实现的本地优先多模态图片检索与整理系统。用户导入自己的图片后，可以用
中文、英文或另一张图片搜索。系统用 SigLIP2 生成图文向量，SQLite 保存业务状态，默认用
低内存 mmap 精确索引检索；React 前端和 FastAPI 后端以单容器部署。公开演示还有会话隔离、
配额和 TTL 的临时图库，并用真实上传—索引—双语检索—删除合同做上线验收。

## 两分钟版本：问题—方案—证据—取舍

**问题**：文件名搜索不理解图片内容，固定 AI Demo 又不能证明真实后端。  
**方案**：双塔模型统一图文空间，SQLite + 可重建 mmap 管状态与检索，local/demo 两种服务
边界兼顾个人工具和公共验证。  
**证据**：100 图本地端到端 500 英文查询 R@1 91.6%；5,000 图精确索引从 3.942 ms 降到
0.363 ms且 Top-10 一致；线上 84 双语查询 Hit@5 95.24%；临时图库 6 条双语 Top-1 全部
通过并验证隔离、缓存和删除。  
**取舍**：Adapter 没超过冻结基线所以不部署；FAISS 更快但默认 mmap 依赖更少且显著降低
100k 向量内存；轻量 SigLIP2 分数不能可靠拒答，所以不冒充概率。

## 高频追问与基于事实的回答

### 为什么 SigLIP2 能做中英文搜图？

双塔将图片和文本映射到同一归一化向量空间，点积用于相对相似度排序。选择 SigLIP2 是因为
同一 100 图、30 条中文配对协议中，CLIP ViT-B/32 R@1 为 3.3%，SigLIP2 为 96.7%；不能把
这个小样本结果解释成对所有中文查询都有效。

### 为什么不用数据库直接存向量并遍历？

SQLite 适合元数据、任务和关系的持久化，但 Python 逐条取向量和点积开销高。连续矩阵把
5,000 图纯索引均值从 3.942 ms 降为 0.363 ms。mmap 让派生向量缓存不必完整常驻；SQLite
仍是可恢复的数据源。

### 为什么不直接上 FAISS / 向量数据库？

FAISS FlatIP 在同一基准为 0.237 ms，确实更快，但当前端到端还包括编码和 HTTP，且 M4
环境记录过 PyTorch/FAISS OpenMP 冲突。统一接口已经保留；在 50k+ 真实图库证明线性扫描
不达预算后，再比较 ANN 的召回、构建、更新和运维成本。

### 你训练了什么，为什么没上线？

训练了冻结 SigLIP2 主干的双塔残差 Adapter，最终配置 198,275 个可训练参数。独立测试英文
R@1 无提升，中文下降 3.34 个百分点，因此结论是 `do_not_deploy`。模型上线标准是独立测试
收益，而不是训练 loss 下降。

### 91.6% 和线上 95.24% 哪个更好？

不能直接比。91.6% 是 100 图、500 英文 caption 的 R@1；95.24% 是 24 图、84 双语类别合同的
Hit@5。候选集、K、查询和相关性定义都不同。面试时应先讲协议，再讲数字。

### 相似度为什么不是置信度？

余弦分数只说明当前查询与候选的相对接近程度，没有经过概率校准。轻量合同中正例最低最高分
与负例最高分区间重叠，因此固定阈值会错拒或错收。页面只展示相对相关性；可选 Qwen3-VL
在小型合同上用 0.40 达到 10/10 负例拒绝，但不能外推为通用阈值。

### 数据如何保护？

本地模式把副本写进专用目录，不修改源文件；写入采用临时文件 + 原子替换。公开临时图库
每会话独立目录/SQLite/索引，响应禁止公共缓存，可主动删除并在 30 分钟后过期。但会话 ID
不是认证，所以不建议上传敏感内容。

### 真实故障是什么？

ModelScope 运行时无法稳定访问 Hugging Face，导致把模型下载放在首请求会失败。修复是构建
阶段缓存公开权重、运行时离线加载，并把模型加载和真实检索纳入健康/部署合同。另一个故障类
是“上传 API 成功但搜索为空”，因此质量门必须验证完整异步链路。

### 你如何测试？

单元/集成层覆盖 API、索引、持久化、任务、会话、标签、重复和评估；前端覆盖智能相册、
相关性和构建产物。CI 做 lint/build/test；发布后再对真实 URL 做双语检索和临时上传合同。
仓库静态可见 81 个 Python 测试函数，但若本次未实际运行，不应回答“81 个全部通过”。

### 如果给你两周继续做？

先扩展版本化查询集和失败切片，再在 50k+ 真实图库测 mmap/HNSW/IVF 的召回、P95、内存和
增量更新；若目标是多人服务，再补认证、对象存储、速率限制和可观测性。不会先无依据拆微服务。

## 代码讲解顺序

1. `src/muselens/api.py`：运行模式和 HTTP 边界；
2. `src/muselens/library.py`：导入事务与业务闭环；
3. `src/muselens/index.py`：算法、缓存和后端接口；
4. `src/muselens/sessions.py`：公开演示安全；
5. `.github/workflows/deploy-modelscope.yml`：交付质量门。

每处只回答“输入是什么、状态在哪里、失败怎么办、证据是什么”。

## 禁止说法

- “准确率 95%”——应说具体协议的 Hit@5；
- “支持 10 万张真实图片”——100k 只是合成向量内存基准；
- “生产级/高并发/百万级”——没有相应负载与运维证据；
- “自己训练了 SigLIP2”——只训练过轻量 Adapter，且未部署；
- “能识别任何图片/能可靠拒答”——均不符合事实；
- “测试全部通过”——除非刚运行并保存了对应版本结果。
